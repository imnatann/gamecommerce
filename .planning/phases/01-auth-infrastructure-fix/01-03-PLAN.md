    - Jika user->isBanned() === true:
      - Panggil Auth::logout() + session()->invalidate() + session()->regenerateToken()
      - JSON request → return 403 dengan message key
      - Web request → redirect ke route('login') dengan flash 'error'
    - Jika user tidak banned: pass through ke $next($request)
    - Return type handle(): Response (Symfony\Component\HttpFoundation\Response)
  </behavior>
  <action>
Buat file sesuai pola exact dari EnsureKycVerified.php. Perhatian khusus:
- Gunakan `Auth::logout()` (facade), bukan `auth()->logout()`
- Flash key adalah 'error' (bukan 'warning') karena ban adalah hard block
- JSON response key adalah 'message' (bukan 'error') — konsisten dengan semua middleware lain
- Middleware TIDAK memblokir user yang tidak login (null user = pass through)

```php
<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Auth;
use Symfony\Component\HttpFoundation\Response;

class EnsureNotBanned
{
    public function handle(Request $request, Closure $next): Response
    {
        $user = $request->user();

        if ($user && $user->isBanned()) {
            Auth::logout();
            $request->session()->invalidate();
            $request->session()->regenerateToken();

            $message = 'Akun Anda telah dinonaktifkan. Hubungi support@gamecommerce.id jika ini kesalahan.';

            if ($request->expectsJson() || $request->is('api/*')) {
                return response()->json(['message' => $message], 403);
            }

            return redirect()->route('login')
                ->with('error', $message);
        }

        return $next($request);
    }
}
```

| Ban check → Auth state | Middleware harus invalidate session, bukan hanya redirect |

## STRIDE Threat Register

| T-01-03-03 | Information Disclosure | Ban reason exposure | accept | Message generik di frontend; ban_reason ada di DB untuk admin review saja — tidak di-expose ke user |
